# 自走設計を全エージェントの基本とする（停止は設計ゲートと不可逆対外操作のみ）

全ハーネスエージェントを**補助設計**（曖昧点で停止して人間に聞く co-pilot）から**自走設計**（決めて・記録して・進み、設計ゲートでのみ人間に晒す）へ作り替える。mid-run 停止が許されるのは (A) 設計されたフェーズゲート（アンカー入力・各レビューゲート）と (B) 不可逆な対外書き込み（Linear 起票・git push・PR merge・課金・外部送信）のみ。他の全曖昧点は decide-record-proceed に変換し、決定は導出ドキュメント／ADR／PR 説明に記録（＝正当化トレイル）してゲートで晒す。

## Status

accepted

## Considered Options

- **却下: 補助設計（既存）**。曖昧点ごとに停止して人間に聞く。人間が隣に居る前提のため、不在だと進まず負荷が消えない。feature-team の Phase 0/3/3.5 escalation、service-designer の一問一答がその実体で、3 週間停滞の設計的原因。

## Consequences

- **feature-team は mid-run 停止ゼロの ticket → PR 自走**になる。Phase 0（薄い spec で停止）→ 仮定を置き PR に明記。Phase 3（レビュー上限 escalate）→ 止めず known issues として PR に明記。Phase 3.5（ADR 候補で停止）→ ADR を書いて PR で晒す。人間の接点は PR レビューと merge（(B)）のみ。
- **「記録」が正当化トレイルを兼ねる**。自走設計とドキュメント追跡可能性は同一機構の裏表（ユーザーが前半で求めた「疑問なら作成ドキュメントを辿る」がここで満たされる）。
- **要件精緻化（grill/brainstorm）も自走化が必要**。人間対話を全要件で行うと破綻するため、self-grill（エージェントが両役を演じ context から自答）とティアリングを後続で設計する。
