# 人間の意思決定を5つの大枠ゲートに限定する（細部は自律）

人間が判断するのは「枠組みがズレうる数少ない瞬間」だけに集約する。細かい方針は自律（self-grill / decide-record-proceed）に倒す。**細かい方針を毎回決めても、大きい枠組みがズレていたら無意味**という原則に基づき、これが autonomous-by-design・self-grill・決定単位ティアリング (Y) の "why"。

大枠ゲートは5つ：

| | ゲート | 性質 | 成果物 |
|---|---|---|---|
| G1 | 企画（アンカー：PRFAQ／デザイン原則／提供形態／マネタイズ） | 枠組みを決める | アンカー（ADR-0002） |
| G2 | プロトタイプ | 枠組みを決める | Claude Design プロトタイプ（ADR-0003/0006） |
| G3 | アーキテクチャ／インフラ | 枠組みを決める | 技術設計ドキュメント |
| G4 | 分解リスト | 枠組みを決める（軽量・骨格のみ） | issue 一括リスト |
| G5 | PR レビュー＋merge | 枠組みを検証する＋唯一の不可逆操作 | PR |

## Status

accepted

## Consequences

- **G4 が人間ゲートなのは「スライス構造＝枠組み決定」だから**。Linear 起票が不可逆だからではない。G4 ではリスト（骨格・依存）だけをレビューし、承認後に自動起票、各 issue の詳細は起票後に self-grill（ADR-0005）が埋める。「枠組みは人間・肉付けは自律」の適用。
- **ADR-0004 (B) の精緻化**：停止する対外書き込みは「不可逆・破壊的・公開」（merge／deploy／課金／外部送信）に限る。**可逆・内部的な書き込み（Linear 起票・branch push・PR を開く）は自律**。issue-decomposer の「起票前に必ず人間承認（省略不可）」は G4 のリストレビューに置き換え、起票自体の停止は廃止。
- G1〜G4 は枠組みを*決める*ゲート、G5 は*検証する*ゲート（既決の枠組みに実装が沿うかを具体コードで最終チェックし、すり抜けた枠組みズレを捕まえる安全網）＋ merge という唯一の不可逆操作。
