# アンカー集合の確定（4点／ペルソナは導出／マネタイズ境界は繰り越し）

人間が事前に握るアンカーを4点に確定する：① PRFAQ ② デザインプリンシプル ③ 提供形態 ④ マネタイズするかどうか（二値）。ペルソナ・使用シーンはアンカーに含めず PRFAQ から自律導出する。マネタイズは二値のみをアンカーとし、無料/有料の境界（守るべきコアループ）はプロトタイプを触りながら決める「繰り越し決定」とする。

## Status

accepted

## Considered Options

- **却下: ペルソナ・使用シーンもアンカーに含める**。的のズレを原理的に防げる利点はあるが、人間の作業最小化という目的に反する。PRFAQ がターゲットと中核シナリオを名指していれば導出可能と判断し、導出側へ。
- **却下: マネタイズの無料/有料境界を事前に固める**。ADR-0004 型の価値判断（コアループに課金しない等）は導出困難だが、ユーザーは「触りながら決める方が良い」と判断。事前ロックではなく繰り越し決定とした。

## Consequences

- ペルソナを PRFAQ から導出するため、アンカーの DoD に「PRFAQ はターゲットと中核シナリオを鋭く名指していること」が必須となる（薄い PRFAQ は的ズレを全派生に伝播させる）。
- プロトタイプは繰り越し決定（例: マネタイズ境界）を**未決の論点として表現でき、それに対して人間が決定を下せる**必要がある。これはプロトタイプの媒体（HTML か否か）に対する要求（十分な対話性）になる。
