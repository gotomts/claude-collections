# 評価ループの差し戻し protocol（全ステージ共通）

共通ステージ形（ADR-0013）は各職種の成果物の直後にプロダクトクリティーク（評価エージェント）を置く。ADR-0011 は「最大3ラウンド差し戻し・findings 付きで該当職種を continuation 再起動・未達は decide-record-proceed」までを決めた。本 ADR はその protocol の機構を確定し、**全ステージ（S1/S3/S4/S5）のクリティークが同一 protocol を使う**ものとする。

## Status

accepted（ADR-0011 の S1 評価ループを全ステージへ一般化・機構を確定）

## 決定

1. **カデンツ＝インクリメンタル＋依存順ゲーティング**：担当職種が成果物を出す → クリティークが即評価 → 合格してから、それに依存する下流職種を起動する。全成果物を作り切ってからの一括（バッチ）評価はしない。これは「上流が評価合格してから下流着手」（ADR-0011）の機構的確定。なおこれは**ステージ内**の前進評価であり、**ステージ間の後工程→前工程フィードバック**（G2→S1 の screen-specs 精緻化・S3→S1 の価格/NFR/実現可否確定・ADR-0013）とは別レイヤで両立する。S1 の成果物は凍結されず生きた成果物。

2. **評価者の起動形＝round 1 fresh → round 2-3 凍結 continuation**：round 1 はクリティークを fresh で立て、独立した初読で**完全な findings マニフェスト**を作る。round 2-3 は同一クリティークを continuation で継続し、**round-1 findings の解消のみ検証**（スコープ凍結＝収束保証）。検証中に新規の重大欠陥を見つけた場合はループを延長せず decide-record-proceed の合図とする。担当職種側は ADR-0011 通り continuation 再起動。独立性は「クリティークが担当職種とは**別職種**」という職種境界で担保し、評価者の毎回 fresh 化には頼らない。

3. **差し戻し予算＝成果物ごとに独立3ラウンド**：各成果物が自前の最大3ラウンドを持つ。ステージ全体のグローバル予算ではない。

4. **上流再オープン（カスケード）はディレクター判断・深さ1**：クリティークが下流成果物の評価中に根本原因が合格済みの上流成果物にあると判定したとき、ディレクターが——軽微（`minor`）は当該成果物で吸収／構造的（`blocker`）な上流欠陥のみ**上流を1段だけ再オープン**（カスケード深さ＝1）。再オープンの差し戻しは**上流成果物自身の残り3ラウンド**から消費する。上流が既に枯渇していれば再オープンせず decide-record-proceed。

5. **finding の構造**：1件の finding ＝ {**対象**（成果物・箇所）／**観点**（①anchor 整合・②真実源整合・③カバレッジ逆引き・④内部一貫性〔品質バー〕）／**重大度**（`blocker`｜`minor`・クリティークが付与・ディレクターが上記4のルーティングに使用）／**根拠**（違反した anchor/真実源/カバレッジ項目の引用）／**期待**（満たすべき状態）／**提案**（任意・採否は担当職種が決める）}。担当職種は finding ごとに **✅解消／➖省略(理由)／⚠️未達(理由)** の3状態で返す。

6. **集計とゲート提示**：ディレクターが3状態を完全性ガード（ADR-0011）に集計し、ゲート（S1 画面一覧レビュー・各ステージの大枠ゲート）で ➖省略／⚠️未達 をギャップレポートとして人間に必ず提示する。黙って欠落が通過しない。

## Considered Options

- **却下：バッチ評価**（全成果物完成後にまとめて評価）。上流欠陥が下流へ伝播してから発覚し、3ラウンド予算内で収束しにくい。
- **却下：評価者を毎ラウンド fresh**（純独立）。毎ラウンド新規 findings を持ち出してゴールが動き収束しない＋コスト増。round 1 fresh → 以降凍結 continuation で独立初読と収束を両取りする。
- **却下：上流を無制限カスケード再オープン**。停止性が保証されない。深さ1＋上流自身の予算消費で有限性を担保する。
- **却下：ステージ全体のグローバル予算**。無関係な成果物同士を結合し、ディレクターの会計が不透明。成果物ごと独立にする。

## Consequences

- **停止性保証**：各成果物 ≤3 ラウンド ＋ カスケード深さ ≤1 ＋ 再オープンは上流自身の予算を消費、で総ロールバックは有限。
- **全ステージ共通**：S1 プロダクトクリティーク／S3・S4 プリンシパルエンジニア／S5 の評価3観点（ADR-0014）が同一 protocol を使う。
- **用語**：corpus を導出/実装する職種を **「担当職種」**と呼ぶ。ディレクター（オーケストレーター／スキル本体）は corpus を自ら書かず、担当職種の起動・findings のルーティング・完全性ガードへの集計を担う。両者を混同する「プロデューサ」等の非正規語は使わない。
