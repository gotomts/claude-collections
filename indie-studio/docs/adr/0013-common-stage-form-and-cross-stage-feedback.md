# ハーネスの共通ステージ形・ステージ間フィードバック・feature-team 廃止

ハーネスは 5 ゲート（G1〜G5）で人間が枠組みを握るステージ列＝**G1 アンカー / S1 企画→ブリーフ / S2 プロトタイプ / S3 技術設計 / S4 分解 / S5 実装**。**S2（Claude Design＝外部ツール）以外の全ステージは共通形「ディレクター（＝スキルを読んだメインセッションの制御役）＋ 職種エージェント群 ＋ 評価エージェント」**で corpus を生む。各ステージは**自律導出＋枠組み判断の対話の混合**（純・自律のステージは無い）。旧 feature-team / fleet（dev×n / rev×n / pr-publisher）は補助設計（停止して人間に聞く）で自走しないため**廃止**し、S5 をこの共通形で作り直す。パイプラインは前方一方向でなく、**定義されたステージ間フィードバック**を持つ。

## Status

accepted（ADR-0011 の S1 編成を全ステージへ一般化。ADR-0009 の fleet 廃止を feature-team 本体へ拡張。ADR-0003/0006 の screen-specs 往復を「核ドキュメント同期」として明確化）

## Considered Options

- **却下: feature-team / fleet を S5 として残す**。Phase 0/3/3.5 で停止して人間に聞く補助設計で自走しない＝ハーネスの目的（自走設計・ADR-0004）に反する。共通形で作り直す。
- **却下: 各ステージを独自構造で作る**。S5（旧 feature-team）が既に「親＋専門エージェント」型で、S1 もこの形に落ちた。全ステージ同型のほうが育てやすい。
- **却下: パイプラインを前方一方向にする**。プロトタイプ・技術見積もりは上流の前提を変える情報を生むため、フィードバック工程が要る。

## Consequences

- **共通ステージ形**：スキル＝ディレクター playbook（人間／将来は横断オーケストレーターが起動すると、メインセッションがそのステージの制御役になる）＋ 職種エージェント（spawn）＋ 評価エージェント（spawn）。**職種エージェントを持たないのは S2 のみ**（Claude Design）。
- **全ステージ＝自律導出＋枠組み対話の混合**。対話点：G1 アンカー全面／S1 画面一覧レビュー／G2 プロトタイプ並列デュアル訂正／S3 スタック・モジュール・参照リポ確認＋S1 繰り越し確定／G3 アーキ・インフラ検証／S4 index.md レビュー＋起票承認／G5 根幹 PR レビュー＋merge。**決めるのは人間・書くのは AI**（ADR-0012）。
- **ステージ間フィードバック（前方一方向でない）**：
  - **G2 → S1**：プロトタイプが画面仕様を**明確化** → その明確さを **screen-specs 核ドキュメントへ書き戻して最新化**（陳腐化・実装時の混乱を防ぐ）。screen-specs ライフサイクル＝①S1 で導出（期待値）→②Claude Design をブリーフ→③プロトタイプで明確化→④書き戻して権威化→⑤S3/S4/S5 が読む。
  - **S3 → S1**：技術見積もり・実現可能性が S1 の繰り越し／⚠️保留（マネタイズ価格・NFR 目標値・機能の実現可否）を確定／修正する。＝S1 corpus は G2 と S3 で訂正される生きた成果物。
- **feature-team / fleet 廃止**：S5 実装ステージを共通形（ディレクター＋実装/レビュー職種＋評価）で作り直す。ADR-0011 で round-limit を「feature-team と同じ」と参照した箇所は構造的後継への参照に読み替える。
- **S3/S4 の成果物（既存スキル準拠・新モデルへ移行）**：
  - S3＝技術スタック／アーキ・ディレクトリ構成／モジュール一覧／接頭辞付き機能一覧 `F-{MODULE}-{連番}`／ドメインモデル（Mermaid）／CLAUDE.md ドラフト／開発の進め方／運用基盤／初期 ADR・CONTEXT.md 種まき。
  - S4＝index.md（垂直スライス＝1 PR・依存・束ね親＝capability・HITL/AFK）＋ issue 詳細（受入条件 BDD/チェックリスト・参照リンク・冪等キー）。
  - 新モデルでは repo-native ＋ 職種エージェント導出へ移行。出力先（Linear / repo）と HITL/AFK・推奨スキル注釈は S5 作り直しに合わせて見直す。
