# スキル分割は人間 handoff 境界で4スキル。横断オーケストレーターは薄い/不要

> ※ 本 ADR の「4 スキル」は、後続 [ADR-0023](0023-design-direction-skill.md) で **5 スキル** に拡張された（S1 と S2 の間に `design-direction` を追加・新 handoff＝画像入力 ＋ 3 問の雰囲気対話）。本文中の 4 スキル列挙はその時点の決定として読み、現行構成は ADR-0023 を参照する。

全自動完走はそもそも目的でない（重要な意思決定は人間・それ以外を AI に任せる）。よってスキルの境界は「1ステージ1スキル」ではなく、**人間の意思決定／環境の切れ目（＋独立チューニング・中断のしやすさ）**で引く。結果、**4スキル**：

1. **アンカー対話＋企画・デザイン導出**：人間と対話してアンカー4点を決め、そこから corpus を自律導出し Claude Design 用ブリーフを作る（→ 人間が claude.ai へ）。
2. **技術設計**：完成プロトタイプを起点にスタック・アーキ・モジュール・ドメインモデル等。
3. **分解**：機能を垂直スライスに割り index.md（骨格）を作成 →〔G4 で人間がリスト承認〕→ **Linear 起票**（承認後・自律）。
4. **実装**：起票済みチケットを実装して PR（→ G5）。

横断オーケストレーターは、自動完走が目的でない以上**複雑な自動連結を持たない**。人間が4スキルを順に起動し、間にゲート／環境境界（特に S2＝claude.ai）が入る。オーケストレーターは「薄い launcher」か、いっそ不要。

## Status

accepted（ADR-0013 の「1ステージ1スキル」想定を「人間 handoff 境界で4スキル」に整理。ROADMAP 項5 のオーケストレーターを「薄い/不要」と確定）

## Considered Options

- **却下: 5スキル（アンカー対話と導出も分ける）**。両者は連続する企画作業で、束ねる方が自然（ユーザー選択）。アンカー確定→そのまま導出。
- **却下: 3スキル（S3＋S4 を束ねる）**。S3 と S4 は別スキルにする方が独立してチューニングでき、間で中断もしやすい（ユーザー）。
- **却下: 自動完走する複雑なオーケストレーター**。全自動は目的でない。人間が要所で起動・判断する。

## Consequences

- 人間の起動列：スキル1 →（claude.ai でプロト・G2）→ スキル2 →（G3）→ スキル3（→ G4 → 起票）→ スキル4（→ G5）。
- **ステージ（G1/S1/S3/S4/S5）と職種ロスターは各スキルの内部構造として残る**。スキル1 は G1（アンカー対話・type-2）＋ S1（自律導出・type-1）の混合。
- 既存の薄実装 `skills/service-derivation`＋`agents/service-deriver`（a509b46）はスキル1 の導出部に相当し、作り直し対象。
- 起票（Linear 書き込み）は分解スキルの末尾＝G4 リスト承認後の自律操作（ADR-0007/0008）。
