# アンカーゲート方式（人間はアンカーのみ決定、以降は自律導出）

人間の意思決定をアンカー（PRFAQ・デザインプリンシプル）に限定し、それ以降の導出ドキュメント生成とプロトタイプ起こしはエージェントが自律的に行う。人間はプロトタイプをレビューし、疑問があれば導出ドキュメントを辿って納得／修正を判断する。アンカーの無いプロトタイプは無価値である一方、派生ドキュメントの一問一答共著が開発を 3 週間単位で停滞させていたため、後者だけを自律化する。

## Status

accepted

## Considered Options

- **却下: プロトタイプ・ファースト（反転）**。アンカー無しで先に視覚プロトタイプを出し、ドキュメントを後から導出する案。PRFAQ・デザインプリンシプルが無いプロトタイプには価値が無いとの判断で却下。
- **却下: 現状の全ステージ対話モード**。派生 12 ページ・画面仕様も一問一答で共著する現行方式。socialcoffeenote で 3 週間かけて約 70 ドキュメントを生成しプロトタイプ 0 という停滞を生んだため却下。

## Consequences

- 自律導出の品質はアンカーの完全性・非曖昧性に強く依存する。アンカーに「自律導出に進んでよい」完了条件（DoD）が必要になる。
- レビュー面がプロトタイプ中心になるため、プロトタイプに現れない決定（データモデル等）の捕捉手段を別途設計する必要がある（後続 ADR）。
