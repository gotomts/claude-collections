# ドキュメントは AI が書く（自律 or 人間との対話駆動）。人間は文書を書かない。Obsidian vault を退役

ハーネスのドキュメントは **2 種類しかない**：(1) **AI が自律的に書く**、(2) **人間との対話から AI が書く**。**人間が手で文書を書くことはない**。アンカー（PRFAQ・デザイン原則・提供形態・マネタイズ二値）は (2)＝人間と対話して AI が書く。それ以外の導出 corpus は (1)＝自律。人間が書く文書が存在しない以上、ADR-0010 が「人間のアンカー執筆面（任意）」として残していた Obsidian vault は役割を失い、このワークフローから**退役**する。

## Status

accepted（ADR-0010 の「Obsidian＝人間のアンカー執筆面（任意）」を改訂し退役。ADR-0001/0002 の「アンカーは人間が対話で決める」を「決めるのは人間・書くのは AI」と明確化）

## Considered Options

- **却下: 人間がアンカーを手書きする**（Obsidian 等で）。人間は対話で**決める**が、文書化は AI が行う方が速く、執筆面の二重管理も消える。「人間は書かない」を原則にする。
- **却下: vault を人間の執筆面として残す（ADR-0010）**。人間が書く文書が無いので、執筆面という存在理由が消える。

## Consequences

- **アンカー捕捉＝G1 の対話スキル**：人間↔AI が対話し、**AI が `docs/discovery/anchors/` にアンカーを書く**。導出ステージ（ADR-0011 の 8 体）はそれを入力に自律で書く。
- **全成果物（anchors 含む）は AI 作**。人間の接点は「対話で決める（G1）」と「ゲートでレビュー（G2 ほか）」だけ。これは ADR-0001/0004（自走設計・人間はゲートのみ）と整合。
- **Obsidian vault はこのワークフローから外れる**（vault の他用途での利用は別問題）。corpus も anchors も repo-native（ADR-0010 の corpus 部分は維持、執筆面部分のみ退役）。
- **アンカー所在解決という論点は消滅**：アンカーは常に repo の `docs/discovery/anchors/` に AI が書くため、vault からの取り込みや所在探索の仕様は不要。
- 本原則はハーネスの全文書（CONTEXT/ADR・導出 corpus・ブリーフ）に適用される。この grill セッション自体も (2) の実例（人間と対話して AI が docs を書く）。
