# ステージ別の職種ロスターと、dev-*/rev-* エージェント廃止・スキル存続

各ステージの導出/実装エージェントは**成果物名でなく「現実の職種」に束ねる**（ADR-0011 の S1 を S3/S4/S5 へ拡張）。あわせて、廃止するのは旧 fleet の **dev-\* / rev-\* / pr-publisher の「エージェント（補助設計の器）」**であって、**スタック/レビューの技術スキル（`flutter-*` / `expo-*` / `dart-*` / `next-best-practices` / `hono` / `vercel-*` 等）は資産として存続**させ、対応する実装/レビュー職種に割り当てる（旧 dev-\* の `skills:` 許可リストの後継）。

## Status

accepted（ADR-0011 の S1 ロスターを S3/S4/S5 へ拡張。ADR-0013 の feature-team/fleet 廃止を「エージェントは廃止・スキルは存続」と精緻化）

## ステージ別ロスター

- **S1 企画→ブリーフ（8体・ADR-0011）**：ディレクター／UX リサーチャー／プロダクトマネージャー／事業・マーケストラテジスト／リスク・法務・品質／プロダクトデザイナー／デザインシステムエンジニア／プロダクトクリティーク（評価）。
- **S2 プロトタイプ**：職種エージェント無し（Claude Design＝外部）。
- **S3 技術設計（6体）**：ディレクター／ソフトウェアアーキテクト（アーキ・モジュール・型・ドメインモデル・F-ID・ユビキタス言語）／テックリード（スタック・CLAUDE.md・開発プロセス・git・テスト戦略）／インフラエンジニア（インフラ・IaC・CI/CD・非機能実現・運用）／セキュリティエンジニア／プリンシパルエンジニア（評価＝設計レビュー）。
- **S4 分解（4体）**：ディレクター／エンジニアリングマネージャー（or テックリード。垂直スライス分解・依存・capability 束ね・HITL/AFK 判定）／QA エンジニア（受入条件 BDD/チェックリスト）／プリンシパルエンジニア（評価＝分解レビュー）。
- **S5 実装（8体）**：ディレクター／フロントエンドエンジニア／バックエンドエンジニア／モバイルエンジニア／インフラエンジニア（器構築・CI/CD・IaC 実装）／コードレビュアー（評価・品質）／セキュリティエンジニア（評価）／パフォーマンスエンジニア（評価・性能影響時のみ）。開発職種は S3 のスタックに応じてディレクターが該当分を並列起動。**QA エージェントは置かず、開発エンジニアがテスト（unit/widget/受入/E2E）を書く**（垂直スライス＝test を貫く原則。E2E 方針はスタック依存で S3 テスト戦略が決める）。網羅性は評価3観点が見る。

## Considered Options

- **却下: 成果物名でエージェントを割る**（persona-agent 等）。現実の職種に束ねる方が凝集が高く割当が自然（ユーザー指定）。
- **却下: dev-\*/rev-\* のスキルも廃止**。スキルはスタック技術知識の資産で、補助設計の器（エージェント）とは別物。存続させ職種に割り当てる。

## Consequences

- S5 の開発職種（フロント/バック/モバイル/インフラ）は S3 のスタック選択で該当分のみ起動（独立スライスは並列）。全部が常に動くわけではない。
- 評価＝各ステージのレビュー職種（S1 プロダクトクリティーク／S3・S4 プリンシパルエンジニア／S5 は品質・セキュリティ・性能の3観点）。
- 実装/レビュー職種は対応する既存スキル群を活用する（職種ごとのスキル割当＝旧 `skills:` 許可リストの後継）。
- ロスターは初期案。運用しながら右サイズ調整しうる。
